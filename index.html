提升SQLite插入速度

发表于 2016-01-07

最近在做一个尝试，将原来直接使用二进制格式保存的文件解析，再用数据库进行管理。做这个尝试的初衷是因为原来格式的文件大小越来越大，已经到达GB的级别，即使只是想查看其中的某个小内容，都需要整体读取，内存占用大。另外，发生修改，保存时，整体重新保存也不是很经济。因此，采取了轻量级的文件型数据库作为数据库解决方案。但在实行过程中，我发现SQLite的插入效率慢得令人发指。查阅一些资料后，将速度提升到了一个可以接受的范围，在此将内容纪录下来。

测试样例

我的设想是：将原有的二进制文件的内容解析出来，分块，然后每一块的数据作为一个数据条目，那么我就能够根据需求，分块存取了。所以我需要在实验之前，建立如下的表：

CREATE TABLE t (ID INTEGER NOT NULL, DATA BLOB)

为了便于测试，我们将问题简化成现有100MB的数据，我们将其分割成100000份1K的小数据块，插入到SQLite数据库中。

暴力插入

最开始直接的思路就是调用SQLite api中直接执行SQL语句的函数：

int sqlite3_exec(
    sqlite3*,                                  /* An open database */
    const char *sql,                           /* SQL to be evaluated */
    int (*callback)(void*,int,char**,char**),  /* Callback function */
    void *,                                    /* 1st argument to callback */
    char **errmsg                              /* Error msg written here */
);

程序代码大致是如下的样子：

int cnt = 100000;
for (int i = 0; i < cnt; i++)
{
    char insert_cmd[100];
    sprintf(insert_cmd, "INSERT INTO t1 VALUES(%d, zeroblob(1000))", i + 1);
    rc = sqlite3_exec(db, insert_cmd, 0, 0, &zErrMsg);
    if (rc != SQLITE_OK)
    {
        std::cout << "SQL error: " + std::string(zErrMsg) << std::endl;
        sqlite3_free(zErrMsg);
        sqlite3_close(db);
        getchar();
        return 1;
    }
}

直接使用INSERT语句插入1KB的数据，总共100MB的数据需要1804.39s，平均每秒执行55.42次插入。

SQLite的事务

调用visual studio的性能分析工具，可以发现时间主要消耗在vbdeCommit()这个函数当中(visual studio真乃神器)

这里就要引出数据库中的一个概念，事务。事务可以理解为数据库的操作单元。没调用sqlite3_exec()一次，就会隐式地开启一次事务。所以之前的代码，每插入一次数据，事务就会反复被开启和提交，造成很大的IO消耗。

好在事务是可以显式开启和关闭的，我们可以将所有数据插入后，再一起提交，则能够大大提高IO效率，从而加快插入速度。

sqlite3_exec(db, "BEGIN", 0, 0, &zErrMsg);
int cnt = 100000;
for (int i = 0; i < cnt; i++)
{
    char insert_cmd[100];
    sprintf(insert_cmd, "INSERT INTO t1 VALUES(%d, zeroblob(1000))", i + 1);
    rc = sqlite3_exec(db, insert_cmd, 0, 0, &zErrMsg);
    if (rc != SQLITE_OK)
    {
        std::cout << "SQL error: " + std::string(zErrMsg) << std::endl;
        sqlite3_free(zErrMsg);
        sqlite3_close(db);
        getchar();
        return 1;
    }
}
sqlite3_exec(db, "COMMIT", 0, 0, &zErrMsg);

经过修改之后，只需要2.12s完成操作，平均每秒47169.8次插入。

##　使用sqlite3_prepare_v2
对于之前的例子，如果我们将数据片段缩小为0.1K，那么100MB的数据则需要分割成1000000份，结果需要7.79s完成操作。因此，执行的插入语句越多，时间消耗越大。

执行一条SQL语句，实际经历了prepared once(准备)，evaluated(求值)，destroyed(销毁)的过程。而sqlite3_prepare_v2就是对SQL语句字符串进行语法分析，并将其编译成目标代码。对于大量数据的插入操作来说，每次都进行准备工作，太浪费时间，其实我们只需要准备一次，就可以重复求值了，这可以节省下语法分析，编译的时间。

sqlite3_exec(db, "BEGIN", 0, 0, &zErrMsg);
int cnt = 1000000;
sqlite3_stmt *stmt;
const char *sql = "INSERT INTO t1 VALUES(?,zeroblob(100))";
sqlite3_prepare_v2(db, sql, strlen(sql), &stmt, 0);
for (int i = 0; i < cnt; i++)
{
    sqlite3_reset(stmt);
    sqlite3_bind_int(stmt, 1, i + 1);
    sqlite3_step(stmt);
}
sqlite3_exec(db, "COMMIT", 0, 0, &zErrMsg);

经测试，同样采取0.1K的数据片段，1000000份数据，上述代码完成操作需要的时间是2.332s，平均每秒能够执行428816次插入操作。

总结

对于大数据的重复插入操作，采取事务+sqlite3_prepare_v2的方式能极大的提升插入效率。

代码

int main()
{
    sqlite3 *db;
    int rc;
    char *zErrMsg = 0;

    rc = sqlite3_open("test.db", &db);
    if (rc)
    {
        std::cout << "Can't open database: " << std::string(sqlite3_errmsg(db)) << std::endl;
        sqlite3_close(db);
        getchar();
        return 1;
    }

    std::string cmd = "CREATE TABLE t1 (ID INTEGER NOT NULL, DATA BLOB)";
    rc = sqlite3_exec(db, cmd.c_str(), 0, 0, &zErrMsg);
    if (rc != SQLITE_OK)
    {
        std::cout << "SQL error: " + std::string(zErrMsg) << std::endl;
        sqlite3_free(zErrMsg);
        sqlite3_close(db);
        getchar();
        return 1;
    }

    clock_t t1 = clock();
    sqlite3_exec(db, "BEGIN", 0, 0, &zErrMsg);
    int cnt = 1000000;
    /*for (int i = 0; i < cnt; i++)
    {
        char insert_cmd[100];
        sprintf(insert_cmd, "INSERT INTO t1 VALUES(%d, zeroblob(100))", i + 1);
        rc = sqlite3_exec(db, insert_cmd, 0, 0, &zErrMsg);
        if (rc != SQLITE_OK)
        {
            std::cout << "SQL error: " + std::string(zErrMsg) << std::endl;
            sqlite3_free(zErrMsg);
            sqlite3_close(db);
            getchar();
            return 1;
        }
    }*/
    sqlite3_stmt *stmt;
    const char *sql = "INSERT INTO t1 VALUES(?,zeroblob(100))";
    sqlite3_prepare_v2(db, sql, strlen(sql), &stmt, 0);
    for (int i = 0; i < cnt; i++)
    {
        sqlite3_reset(stmt);
        sqlite3_bind_int(stmt, 1, i + 1);
        sqlite3_step(stmt);
    }
    rc = sqlite3_exec(db, "COMMIT", 0, 0, &zErrMsg);
    clock_t t2 = clock();
    sqlite3_close(db);

    double time = double(t2 - t1) / CLOCKS_PER_SEC;
    std::cout << "cost time: " << time << "s." << std::endl;
    std::cout << "insert speed: " << cnt / time << "cmd/s." << std::endl;
    getchar();
    return 0;
}
